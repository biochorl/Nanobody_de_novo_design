About

From a target structure to a designed binder

This material was used during the Nanobody Workshop (22–26 Sep 2025) at the University of Nova Gorica. It is built for teaching and demonstration on free Colab GPUs — adequate to learn the concepts, not to scale a production screening.

📱

Runs on your phone

Every notebook has a “Mobile mode” switch at the very top. Turn it on and all the input files for that step are downloaded automatically from this repository — so you can follow the whole workflow from Google Colab without any local file system.

The pipeline

Four steps, four notebooks

Run them in order. Each opens in Google Colab and produces the inputs for the next.

1

Epitope prediction & CDR identification

DiscoTope-3.0 · Nanocdr-X

Prepare everything IgGM needs: predict B-cell epitope hotspots on the antigen and mask the nanobody CDRs to redesign.

  • DiscoTope-3.0 epitope hotspots → 0-based indices for IgGM
  • Builds IgGM_input.fasta (CDRs = X) and antigen_A.pdb
⬇ epitope/CDR annotations ⬇ IgGM_input.fasta
2

De novo CDR design & side-chain refinement

IgGM · PIPPack

IgGM co-designs the sequence and structure of the CDRs against the antigen — a full all-atom complex — then PIPPack repacks the side chains in the antigen context. Replaces the old RFantibody + ProteinMPNN + PIPPack chain.

  • Optional random sampling of CDR3 length per design
  • Exports cdrs_annotation.json with the exact CDR mask of each design
⬇ PIPPack_outputs.zip facsimile ⬇ cdrs_annotation.json facsimile
3

CDR sequence optimization (in complex context)

AntiFold

Re-design the CDR sequences with a CDR-specialized inverse-folding model, with the antigen frozen as structural context. Uses the per-design cdrs_annotation.json CDR mask, so only the variable-length CDRs are touched.

⬇ AntiFold_Best_Designs.zip facsimile
🔑

Before step 4 — get a free BioHub API token

BioHub · one-time setup

Step 4 folds your designs with ESMFold2 through the BioHub API, which needs a free API token. Do this once:

  • Open biohub.ai and sign in (free account).
  • Go to Developer Console → API keys: biohub.ai/developer-console/api-keys.
  • Click Create API key, copy it, and paste it into the biohub_token field of the step-4 notebook.

⚠️ Keep the token private — don’t share a notebook with the token filled in.

4

Blind validation by structure prediction

ESMFold2 · BioHub

Fold the redesigned complex sequence from scratch (no design coordinates as a guide), align it to the design for an RMSD, and report pLDDT / pTM / ipTM — the key interface confidence metric.

Beyond the tutorial

Screening & further validation

This 4-step pipeline teaches the concepts. For a real campaign you would generate ≈1000 in-silico designs and filter them hard — expect ~95% to fail. Validation usually combines several orthogonal lines of evidence:

Deep-learning interaction confidence

  • Re-predict the complex with AF2 / AF-Multimer / AF3 without templates and check consistency with the design
  • Flexibility-aware interface scores: Local Interaction Score (LIS), ipSAE, ipTM/pTM

Empirical / physics scores

  • Protein–protein docking: HADDOCK, Rosetta-ddG, Fold-X
  • Interface geometry: dG/dSASA, packstat, buried unsatisfied H-bonds

Physics-based assessment

  • Molecular dynamics (>100 ns) for complex stability (GROMACS, AMBER)
  • Enhanced sampling for the binding free-energy landscape (metadynamics / PLUMED)

Developability

  • Solubility, thermostability, aggregation propensity
  • Off-target / polyreactivity risk — compare against the parent scaffold

🚀 The field is getting much easier

Generative protein design is becoming far more accessible, increasingly through ready-made web interfaces that hide all the infrastructure. A good example is BoltzGen, usable directly through the BoltzLab platform — we will go through it quickly to show how easy it now is to design a binder from a browser.
